# Product Detail Page Auto Skill

## 목적

상품의 기본 정보, 레퍼런스 이미지, 실제 상품 이미지를 입력받아  
상세페이지 기획안, 섹션별 디자인 이미지, 최종 ZIP 파일까지 자동 생성하는 Skill입니다.

이 Skill은 네이버 스마트스토어, 쿠팡, 자사몰, 오픈마켓용 상품 상세페이지 제작에 사용합니다.

---

## 전체 진행 순서

1. 상품 기본 정보 입력
2. 레퍼런스 이미지와 실제 상품 이미지 입력
3. 상품 정보에 맞는 카피라이팅 방향과 이미지 자료 수집
4. 상세페이지 기획안 생성
5. 기획안을 바탕으로 상세페이지 디자인 방향 확정
6. 섹션별 이미지를 한 장씩 순차 생성
7. 생성된 섹션 이미지를 사용자에게 보여주기
8. 최종 이미지 파일을 ZIP으로 묶어 다운로드 제공

---

## 기본 출력 기준

- 상세페이지 권장 가로 사이즈: 860px
- 섹션별 이미지 권장 사이즈: 860px × 1200px 이상
- 전체 상세페이지 예상 길이: 최소 8,000px ~ 12,000px
- 섹션 수: 기본 8개
- 텍스트는 이미지 안에 과도하게 넣지 않는다
- 실제 상품 이미지를 우선 사용한다
- 레퍼런스 이미지는 분위기, 구도, 연출 방향 참고용으로만 사용한다
- 상품 정보와 다르게 과장 표현하지 않는다
- 식품, 건강, 기능성 상품은 법적 문제가 될 수 있는 효능 표현을 피한다

---

## 입력받아야 할 정보

사용자에게 아래 정보를 요청합니다.

```md
상품명:
브랜드명:
상품 카테고리:
판매 채널:
타깃 고객:
핵심 장점 3~5개:
상세 특징:
상품 구성:
옵션:
가격대:
신뢰 요소:
주의사항:
원하는 분위기:
레퍼런스 이미지:
실제 상품 이미지:
```

입력값이 부족하면 임의로 진행하지 말고, 부족한 항목을 먼저 확인합니다.  
단, 사용자가 “알아서 진행” 또는 “기본값으로 진행”이라고 하면 아래 기본값을 사용합니다.

```md
판매 채널: 스마트스토어
타깃 고객: 일반 소비자
원하는 분위기: 실용형 / 강한 판매형
섹션 수: 8개
이미지 사이즈: 860px × 1200px 이상
```

## 상세페이지 기본 섹션 구조

기본 섹션은 아래 순서를 사용합니다.

1. HERO
   - 첫인상, 상품명, 핵심 USP
   - 상품 이미지 중심
   - 상단 30% 안에 핵심 판매 포인트 배치

2. PROBLEM
   - 고객이 겪는 불편함 또는 구매 이유
   - 문제 제기형 카피

3. BENEFIT
   - 상품의 핵심 장점 3~5개
   - 카드형 또는 아이콘형 구성

4. USE CASE
   - 사용 상황, 섭취 상황, 착용 상황, 활용 장면
   - 라이프스타일 이미지 중심

5. DETAIL
   - 소재, 원물, 디테일, 기능, 구성 설명
   - 실제 상품 컷과 클로즈업 중심

6. COMPARISON
   - 일반 상품 대비 장점
   - 비교표 또는 차별점 정리
   - 과장 비교 금지

7. SPEC
   - 용량, 사이즈, 구성, 보관법, 옵션 정보
   - 구매 전 확인 정보

8. NOTICE / CTA
   - 배송, 보관, 주의사항, 구매 유도 문구
   - 브랜드 신뢰감 마무리

---

## 카피라이팅 규칙

- 첫 문장은 강하게 쓴다
- 상품명보다 고객 이익을 먼저 보여준다
- 단순 설명보다 구매 이유를 만든다
- 반복 표현을 피한다
- “최고”, “완벽”, “100% 효과” 같은 과장 표현은 사용하지 않는다
- 식품은 질병 예방, 치료, 의학적 효능 표현을 쓰지 않는다
- 건강 관련 표현은 “건강간식”, “부담 없는 간식”, “가볍게 즐기는” 수준으로 제한한다
- 고객이 바로 이해할 수 있는 쉬운 문장을 사용한다

---

## 이미지 생성 규칙

섹션 이미지는 한 번에 전체를 생성하지 않고, 반드시 섹션별로 순차 생성합니다.

각 섹션 생성 시 아래 정보를 포함합니다.

```md
섹션명:
섹션 목적:
사용할 실제 상품 이미지:
참고할 레퍼런스 이미지:
배경 분위기:
레이아웃:
포함할 카피:
포함하면 안 되는 요소:
이미지 사이즈:
```

### 이미지 생성 기본 조건

- 가로 860px 기준 상세페이지형 이미지
- 세로 1200px 이상
- 쇼핑몰 상세페이지 스타일
- 실제 상품 중심
- 깔끔한 배경
- 고급스럽지만 정보 전달이 명확한 구성
- 텍스트는 가독성 있게 배치
- 워터마크 금지
- 의미 없는 장식 금지
- 상품 형태 왜곡 금지
- 실제 상품 이미지와 다른 제품처럼 보이게 만들지 않기

---

## 디자인 규칙

- 첫 화면에서 핵심 USP가 보여야 한다
- 섹션마다 역할이 달라야 한다
- 같은 장점을 여러 섹션에서 반복하지 않는다
- 상품 사진, 카피, 여백의 균형을 맞춘다
- 섹션별 분위기는 통일하되 구도는 다르게 구성한다
- 식품은 식욕을 돋우는 조명과 질감 표현을 우선한다
- 의류는 착용 핏, 원단감, 컬러감을 우선한다
- 생활용품은 사용 전후, 기능 설명, 실사용 장면을 우선한다

---

## 출력 파일명 규칙

생성 이미지 파일명은 아래 형식을 사용합니다.

```txt
01_hero.png
02_problem.png
03_benefit.png
04_use_case.png
05_detail.png
06_comparison.png
07_spec.png
08_notice_cta.png
```

최종 ZIP 파일명은 아래 형식을 사용합니다.

```txt
상품명_detail_page_images.zip
```

## 최종 출력 형식

사용자에게 아래 순서로 제공합니다.

1. 상세페이지 기획안 요약
2. 섹션별 생성 이미지 미리보기
3. 생성된 이미지 파일 목록
4. ZIP 다운로드 링크

---

## 실행 프롬프트

사용자가 “상세페이지 자동 제작 시작”이라고 입력하면 아래 순서로 진행합니다.

1. 상품 입력 템플릿을 제시한다.
2. 사용자가 입력한 정보를 검토한다.
3. 부족한 정보가 있으면 질문한다.
4. 충분하면 상세페이지 기획안을 생성한다.
5. 섹션별 이미지 생성 순서를 안내한다.
6. HERO 섹션부터 순차적으로 이미지를 생성한다.
7. 각 섹션 생성 후 결과를 보여준다.
8. 모든 섹션 생성이 끝나면 ZIP 파일로 묶어 제공한다.

---

## 사용자 시작 문장 예시

```txt
상세페이지 자동 제작 시작
```

또는

```txt
상품명: 오육포
브랜드명: 건어물점빵
상품 카테고리: 식품 > 수산품 > 건어물
핵심 장점: 술안주, 건강간식, 육포보다 부드러운 오징어육포
원하는 분위기: 실용형 / 강한 판매형
실제 상품 이미지 첨부함
레퍼런스 이미지 첨부함
상세페이지 자동 제작해줘
```
